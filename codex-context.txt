You assist with development of the “varmapack” C library (varmasim).  
Always obey the rules below.

---------------------------
C CODING STYLE (STRICT)
---------------------------
- Standard: ISO C (C11 only). No compiler extensions or attributes.  
- Indent: 2 spaces.  
- Comments: // only.  
- Pointer style: int *x.  
- Brace style: Stroustrup; else on its own line.  
- Max line width: 90 chars.  
- Prefer i++ to ++i in for-loops.  
- Multiplication: no spaces around *.  
- Use bool/true/false; prefer 0 to NULL.  
- Public API: snake_case with prefix varmapack_.  
- Internal helpers: static or static inline, no prefix.  
- No min/max macros. Use fmin/fmax for doubles; define static inline imin/imax for ints.  
- No macros except include guards.  

---------------------------
PROJECT ARCHITECTURE
---------------------------
- Public API declared in varmapack.h only (with C++ guards).  
- Internal code lives in src/ and internal headers (static inline only).  
- No visibility attributes; library is pure C.  
- Layout:
    src/        implementation + internal headers  
    tests/      unit tests  
    examples/   sample programs  
- Meson used for builds.

---------------------------
PROJECT STATUS
---------------------------
- R interface is on hold.  
- Goal: clean, portable C library with consistent naming and internal structure.  
- Only varmapack_* symbols are exported.

---------------------------
OUTPUT RULES
---------------------------
- All C code must follow the style above exactly.  
- Use 2-space indentation and Stroustrup braces.  
- No commentary about these instructions in output.  
- Tone: precise, technical, concise.
